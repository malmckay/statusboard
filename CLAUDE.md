## Commands

```bash
# Install dependencies
npm install

# Local development (seeds D1 local DB, then starts dev server)
pnpm dev

# Type-check and validate deploy (dry-run)
npm run check

# Generate Cloudflare Worker types
npm run cf-typegen

# Deploy to production (automatically runs migrations first via predeploy hook)
npm run deploy

# Apply migrations to local D1 only
npm run seedLocalD1
```

No test framework is configured.

## Architecture

This is a minimal Cloudflare Workers project backed by a Cloudflare D1 (SQLite) database.

**Entry point:** `src/index.ts` — exports a single `fetch` handler that queries the D1 database via `env.DB` (a `D1Database` binding) and returns an HTML response.

**Rendering:** `src/renderHtml.ts` — pure function that wraps content in an HTML template.

**Database:** `migrations/` contains SQL migrations applied via Wrangler. The D1 binding is named `DB` in both `wrangler.json` and the generated `Env` type in `worker-configuration.d.ts`.

**Type generation:** `worker-configuration.d.ts` is auto-generated by `wrangler types` — do not edit manually.

**D1 binding name:** In `wrangler.json`, the database binding is `"DB"` (not the database name `"d1-statusboard-database"`). Wrangler commands for local vs remote use `DB` as the binding identifier (e.g., `wrangler d1 migrations apply DB --local`).
